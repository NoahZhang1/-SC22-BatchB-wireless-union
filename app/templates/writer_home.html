The Project

Generate your fake headline with any prompt

Trained with a kaggle dataset of over 200,000 news headline in different categories and a GPT-2 based aitextgen model with 124 million parameters, we built a model that can generate fake headlines that are both interesting and qualitatively convincing

Headlines and article - we can do both!

We achieved not only fake headline generation but also fake news content generation with generated fake headline being fed as input, made possible by grover mode



Interactive Website

We made an interactive website for users to enter prompt for the news headline and allows user to get corresponding news content within one single page, which was all made possible by deploying chunky models backend with multiprocessing




Extensions - discord bot and more!

We deployed our model to a discord bot where the bot automatically generate fake headlines accordingly to the prompt message that user sent in the channel

Dataset

Categorical Fake News Dataset

Categorical News Dataset from Kaggle

We found our dataset on Kaggle, where the dataset contains over 200,000 pieces of news headline with 6 different categories

Preprocessing

We used nltk and string library to do dataset preprocessing and split dataset accordingly to their corresponding categories

Models and Training

How We Made the Models

GPT-2 based aitextgen

We used aitextgen for headline generation. All models were trained by finetuning the 124M pre-trained language model


Grover

Grover is a fake news generation and detection model published by University of Washington in Dec 2020. Trained with news dataset of 120 gigabyte large, the grover can achieve SOTA performance in generating fake news content




Process

Finetuing the aitextgen model with news headline dataset with different categories usually take up to 30 minutes. With generated headline feed as input, we can utilize the grover model for content generation




Problem-Solving

Due to the complexity of the grover model, generation of news content with CPU is time-consuming - a successful generation usually takes around 2 minutes, which consistently causes 502 timeout. With that in mind, we utilized multiprocessing to allow different tasks to be held at the same time.
